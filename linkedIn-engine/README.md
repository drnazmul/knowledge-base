## üéØ Best Practices

### When Adding New Content
‚úÖ **DO:**
- Be specific (concrete examples, numbers, dates)
- Organize clearly (use headers and lists)
- Add context (why this matters)
- Include "last updated" date
- Write commit messages describing changes

‚ùå **DON'T:**
- Add duplicate information
- Be vague or generic
- Forget to organize properly
- Skip commit messages

---

### Version Control Tips

**Commit Messages:**
- Good: "Added 5 new SPME topics to topics-and-ideas.md"
- Good: "Updated story-bank with Deciem cost-saving story"
- Bad: "Update"
- Bad: "Changes"

**When to Commit:**
- After adding new ideas
- After completing significant updates
- Before switching tasks
- End of work session

## üÜò Troubleshooting

**Problem:** Can't edit file on GitHub  
**Solution:** Make sure you're logged in and it's your repository

**Problem:** Changes not showing  
**Solution:** Refresh page, check if commit succeeded

**Problem:** Lost track of changes  
**Solution:** Click "History" on any file to see all versions

**Problem:** Want to undo a change  
**Solution:** Go to History ‚Üí Click on previous version ‚Üí Copy that content ‚Üí Edit current file ‚Üí Paste old version

**Problem:** File conflicts  
**Solution:** GitHub web interface avoids this. If it happens, save both versions, merge manually
